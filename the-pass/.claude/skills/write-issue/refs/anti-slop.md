# 反 AI slop + 最高原則（全編輯共用底線）

## 最高原則：事實不可扭曲（最優先，凌駕一切）

| 規則 | 說明 |
|------|------|
| **引號 = 實話** | 原文裡一定要有人真的說過這句；**不可虛構引述**。 |
| **數字 = 原文** | 數據必須來自原始信源，不誇大、不縮小、不亂四捨五入。**別自己從原文年份推算數字**：原文寫「2007 年確診」就別寫成「十八年前」——那是推算、還可能算錯；要嘛照給年份、要嘛寫「多年前」。 |
| **情境可查證** | 描述的場景要基於報導內容；用「想像／如果」時要明示是假設。 |
| **角度 ≠ 篡改** | 選角度是取捨，不是改變事件意思。 |
| **不知道就說不知道** | 原文沒有的資訊不要補；寧可少寫一句，也不多編一句。 |

> 觀點不是扭曲：選題、角度、提問本身就是觀點（編輯本職）。溫度來自角度與提問，**不來自捏造**。
> 因此寫作前必須讀**抓回來的全文**，所有引號與數字都要能對回原文。
>
> **標示**：不確定的資訊用「據稱／根據報導」標明；引用數據時標注來源。

## 禁止開頭

「在當今快速變化…」「隨著…飛速發展…」「近年來…」「想像一個世界…」「讓我們一起…」

## 禁止詞彙

令人震驚的、革命性的、劃時代的、值得注意的是、此外、再者、不僅如此、毫無疑問、眾所周知、未來充滿無限可能、拭目以待。

## 禁止結構

- 萬用三段式（背景 → 三點重點 → 展望未來）。
- 假平衡（「一方面…另一方面…」）→ 改成兩個並陳的短句：「AI 讓廚房更安全。但廚師不太開心。兩件事同時真實。」
- 條列式（這是 AI 的母語）→ 預設改敘事式（像說話）。**唯一例外：技術規格／具體數據可用短列表，但每列表 ≤5 項、一篇最多 1–2 個。**
- **數據轟炸**（一段堆 3+ 個統計／百分比）→ 跟條列一樣是 AI 母語。一段留 **1–2 個最有感的數字**，其餘融成一句話或捨棄——數字服務一個論點，不是當市場報告。

## 四個寫作原則

1. **具體取代抽象**：不說「提升效率」，說「食材浪費從 30 公斤降到 8 公斤」。
2. **畫面取代說明**：不說「AI 攝影機能辨識衛生違規」，說「帽子還沒戴好，手機就震了」。
3. **對話感取代陳述感**：不說「技術導入引發討論」，說「如果你是那個廚師，你 OK 嗎？」
4. **承認取代全知**：不說「將重塑未來」，說「這會怎麼發展，老實說沒人知道」。

## 通用禁忌

禁 emoji（Passe 標題的國旗例外）、禁新聞稿用語（據悉、引發關注）、禁模糊量詞（很多、大量、顯著）、禁驚嘆號、禁替受訪者下結論（他沒說過的感受不要寫）。

## 語言（全域）

- 用**繁體中文（台灣用語）**。
- 外國品牌名、人名**保留原文，首次出現時附中文翻譯**。
- 韓文、日文專有名詞保留原文，附中文說明。
- **數字**：硬數據／統計／百分比／年份／金額用**阿拉伯數字**（55%、2021 年、415 億韓元、16.5 度）；純修辭／概數可用國字（一場爭執、半小時、三塊大陸）。別整段把數據寫成國字——讀起來像舊公文。
