# 總編輯審核清單（7 項）

所有編輯寫完後才介入，讀完整草稿、逐項對照。**只標問題、給修改方向，不直接改寫**（在 /write-issue 單次流程裡，由你扮演總編標出問題→自己回頭修→再過一次）。全通過才標「可發佈」。

1. **事實核查** — 數字、公司名、人名、職稱與原文一致？引述真的出現在原文？未被原文支持的陳述一律標記。（對照抓回來的全文。）
2. **標題品味** — 有「先找到人、再提技術」？有沒有踩禁句「[公司] 用 AI [做某事]，[數字]」？
3. **AI 味檢測** — 有沒有條列式、禁用詞（令人震驚的／劃時代的／此外／再者…）、機器腔（文法對但沒人味）？
4. **重複檢查** — 本期幾則主題會不會太像？跟上期是否重複國家或應用類型？Passe 有沒有重複 Mise 的角度？
5. **整期節奏** — 2 長 + 3–6 短的節奏對嗎？國家／來源夠多元嗎？是不是整期都在講「效率提升」而少了人文面？
6. **品牌一致性** — 語氣有沒有偏離 The Pass（變成 TechCrunch 腔）？有沒有不小心下結論或給建議？
7. **輕量 SEO** — 標題含讀者會搜的核心概念？前兩段能讓人/搜尋引擎抓到主題？
   - **最高原則：內容有趣永遠優先於 SEO。** 若 SEO 犧牲了標題人味或可讀性，一律不採納。

**輸出**：每篇標「✅ 可發佈」或「⚠️ 待修：<哪裡><違反哪條><建議方向>」。流程結尾附一份審核摘要。
