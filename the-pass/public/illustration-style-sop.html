換插圖風格

把「換一套插圖風格」變成一件你能隨時做的事。重點不是某個風格好不好，是讓風格變成可替換的單元。

核心觀念：換風格 ＝ 換一個「Style Profile」。 Profile 是資料、不是 skill——生圖的流程 /publish-issue 永遠不變，它只是去讀「現役 profile」。換風格 = 換它讀哪個 profile。這跟編輯那套是同一個道理：
可替換的資料模組穩定的流程
編輯 ＝ soul／voices／memory/write-issue
風格 ＝ style.md ／ anchors ／ validation/publish-issue
一個 Profile ＝ 三部分（show ／ tell ／ prove）
① 錨點圖 anchors/你給的 2–4 張代表性原圖。生圖時當 input_image 餵進去「定錨」——純文字釘不住風格，這幾張才鎖得住。the show
② 規格 style.md從圖抽出的文字規格：配色／線條／質感／構圖／造形／情緒／禁忌，外加每次生圖要貼的提示詞片段＋negative。一個檔講完「這風格是什麼、怎麼叫出來」。the tell
③ 驗證 validation/用這 profile 實際生成的測試圖＋一份逐面向打分報告。證明它真的能重現，不只是「規格看起來對」。the prove
提示詞片段和 negative 不是獨立檔，就寫在 style.md 裡。真正落在磁碟上的是三樣：一個 style.md ＋ 兩個資料夾 anchors/、validation/。來源不限——插畫、照片、截圖、mood board 都能抽。
跑完 /style-extract，你會拿到一個資料夾

三部分不是抽象概念，是磁碟上一個真的資料夾。skill 跑完，profile 存進共用風格庫 ~/style-lab/（跨專案重用），長這樣：

# ~/style-lab/profiles/<風格名>/    一個自帶一切的可替換單元
style.md                  # 規格 ＋ 提示詞片段 ＋ negative ＋ 來源/日期/驗證摘要
anchors/                  # 你給的原始參考圖（複製進來，2–4 張）
  anchor-1.jpg
  anchor-2.jpg
validation/               # 用這 profile 生的測試圖 ＋ 打分報告
  test-chef.png
  test-farmer.png
  test-ramen.png
  report.md               # 逐面向 0–5 分、哪裡飄掉、能不能重現
項目是什麼為什麼要它
style.md文字規格＋提示詞片段＋negative＋一行驗證摘要tell。把風格寫成可讀、可貼的設定。提示詞和 negative 都收在這一檔，不另外散。
anchors/你給的 2–4 張原始參考圖（複製進來）show。純文字釘不住具體風格，生圖把這幾張當 input_image 餵進去才鎖得住。profile 一定要帶錨點。
validation/AI 用這 profile 生成的測試圖 ＋ report.mdprove。skill 的價值所在。每張測試圖由獨立的驗證 subagent 對著原圖逐面向打 0–5 分（palette／line／texture／composition…）、標出哪裡飄掉；沒過就改提示詞重生重驗。留下的＝「驗證過、能重現」的證據。
別把 anchors 跟 validation 搞混：anchors/ 是你給的原圖（輸入、風格的源頭、題目）；validation/ 是AI 生出來的測試圖（輸出、拿去對照原圖驗收、答卷）。一個是題目、一個是答卷——對照打分，才知道這 profile 穩不穩。
為什麼非驗證不可？眼睛會被騙、分數不會。實測 mono-ink 那組：肉眼看「一次就很像」，驗證 subagent 逐面向一打，line 面向三張全是 2／5（少了毛筆粗細變化＋實心黑塊）——改完提示詞重生，line 才跳到 4–5、兩張升上 reproduced。交付「驗證過」的，比交付「看起來對」的可靠太多。
換風格 = 兩步
1
跑 /style-extract，得到 profile skill
丟一組參考圖 → skill 自己跑完整迴圈：多面向分析 → 規格 gate（你確認）→ 生測試圖 → 驗證 subagent 逐面向打分 →（飄掉就改提示詞、重生、重驗）。產出一個驗證過的 profile（style.md ＋ 錨點圖 ＋ validation 證據），存進共用庫 ~/style-lab/。
2
導入專案，換現役 Claude
把選好的 profile 設為某專案的現役風格：該專案出圖 skill（如 The Pass /publish-issue）指過去、更新人看的 illustration-guide。以後那個專案生圖自動走新風格。
所以是兩步：① /style-extract 產一個驗證過的 profile（跨專案、可重用）→ ② 哪個專案要用，就把它導入、換現役。換風格 = 重跑這兩步。
範例：profile 裡的 style.md（節錄）

這是上面那個資料夾裡 style.md 的長相——把一組大膽 pop 插畫抽成的文字規格（同個資料夾裡還躺著 anchors/ 原圖和 validation/ 測試圖）：

# Style Profile · Bold Pop / Riso（範例）
配色：單一高彩度平塗底（熱粉／電光藍／鎘紅／檸檬黃／teal）＋黑，一圖一主色
線條：粗黑、手繪、帶拙味的墨線
質感：riso／絹印顆粒、蠟筆填色、半調點點、看得見筆觸
造形：naive 孩童感，臉誇張，膚色用非自然 pop 色
構圖：大膽中心人物 ‧或‧ 漂浮元素拼貼；平塗單色地、無景深
情緒：戲謔、街頭、音樂海報／zine
錨點：（該風格的 2–4 張代表圖）
禁忌：寫實膚色／3D／漸層／照片感／來源殘留字幕
提示詞片段：bold naive folk-pop, riso/screenprint, thick black ink,
  crayon grain + halftone, flat single saturated background,
  stylized pop-colour faces (no realistic skin), 16:9

↑ 用這個 profile 實跑出來的 3 張（不同主題：廚房／田裡／拉麵）——同一套風格、跨內容都重現。經 /style-extract 的驗證迴圈打分確認（這套大膽風 The Pass 不採用，純示範「換風格」流程跑得通）。

style.md 範本（直接複製去填）
# Style Profile · {風格名}

一句話：{一句抓住這個風格的調}
配色：{主色 + hex + 用法；平塗還漸層；有沒有特殊質感}
線條：{粗細／手感／要與不要}
質感：{印刷感？顆粒？網點？}
人物／造形：{怎麼畫人、臉、比例；可見與不可見}
構圖：{主角是誰、留白多少、比例 16:9}
品牌固定元素：{出菜框＋桌鈴 要不要保留／怎麼融入這風格}
禁忌：{這風格絕不要出現的東西}
錨點圖：{2–4 張代表圖路徑}
提示詞片段：{每次生圖貼的那段}
Negative：{要避免的清單}
範本檔也在 repo：docs/illustration/styles/risograph/style.md 就是現役風格的完整實例，可直接照抄結構。
