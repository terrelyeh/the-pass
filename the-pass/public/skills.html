編輯室的三個 Skill

整條出刊由三個 skill 接力完成，中間夾「人定稿」。每個 skill 把一套方法固化成一個指令——機器跑流程、人定品味。
選題→寫稿→你定稿→發佈

選題

/selection-report

每期把全球來源抓回來、篩選評分，產出一份「選題報告」讓你拍板要寫哪幾篇。把「憑感覺挑題」變成有方法、可重複。

流程

① 抓取 — 從來源庫（約 30 個聚焦食物的來源）抓回最新文章
↓
② 去重 — URL ＋ 標題相似度，折疊重複的
↓
③ Haiku 子代理粗篩 — 快速刷掉明顯不合的，留下候選池
↓
④ Claude 當總編評分 — 五面向（食物相關／新鮮獨家／人味故事／觀點／可讀性）逐篇評分 ＋ 編輯路由
↓
⑤ 庫存跨期競爭 — 上期沒選上的合格稿一起競爭（30 天保鮮）
↓
⑥ 出選題報告 — 建議出刊 ＋ 完整候選 ＋ 每篇三編輯切角，你拍板

特點

  • 零 API key——在你本機 Claude Code 跑，不用金鑰、不用伺服器。
  • 雙輸出——選題報告同時上網頁（團隊看、可切換歷期）＋ 進 Obsidian（你存檔）。
  • 跨期去重——出過、撈過的記起來，下期不重複。

輸入 → 輸出

來源庫 sources.ts→選題報告（網頁＋Obsidian）selected.json

怎麼用

跟我說「跑選題 / 這期選什麼」，或在 Claude Code 觸發 /selection-report，就自動跑完整套。
看最新選題報告 ↗篩選機制設計 ↗

寫稿

/write-issue

把選題拍板的稿，寫成整期草稿（2 長文＋快訊＋1 提問）。四個編輯分身各用自己的筆調寫，AI 起草、人定稿。

流程（含 3 個人拍板關卡）

① 抓全文查證 — 對每篇抓原文當素材，所有引號與數字都要對得回原文
↓
② Mise 寫頭條長文 — 場景式開頭、從人切入
🚦 GATE 1 · 你校準筆調 — 筆調對嗎？切角 OK 嗎？對了才往下
↓
③ Mise 寫第二長文　‖　Passe 寫快訊 — 並行
↓
④ Fumet 從長文提煉一個提問 — 留一個真開放的問題
↓
⑤ 總編獨立審 7 項 ＋ 逐條對事實
🚦 GATE 2 · 事實對不上 → 停下問你
🚦 GATE 3 · 發佈前你逐篇定奪 — 採用／改／退 → 草稿進 Obsidian

為什麼是四個獨立分身

同一個 AI 在同一個腦袋裡寫完三種編輯，筆調會互相滲透。所以每位編輯各開一個分身，腦袋裡只裝自己的三層大腦——Soul（人格／為什麼這樣寫）＋ Memory（學到的準則／寫過的主題）＋ Voices（怎麼寫）。總編也獨立，純挑錯不自審。

守門原則

  • 事實不可扭曲——引號＝原文真有人說過、數字＝原文，不捏造。
  • AI 起草、人定稿——三個 gate 都停下來問你，AI 不替你做最終決定。
  • 筆調隔離——四個分身各自獨立，筆調不互相污染。

發佈

/publish-issue

把你定稿的內容，做成 thepass.cc 的 issue 網頁，含長文配圖（紐約客式概念插畫）。在你離線潤定稿之後跑。

流程

① 讀定稿 — 你潤好的出刊草稿，內文不改、只做發佈
↓
② 長文配圖（每篇一張，快訊不配） — 先想一個紐約客式諷刺概念 → 以 demo 圖為風格錨點 → nanobanana ＋ Codex 雙模型生候選
🚦 你挑圖 — 看候選頁挑每篇要哪版
↓
③ 渲染 issue 頁 — 標題下方放封面圖、共用 issue.css
↓
④ 更新配圖總覽 ＋ 部署

配圖原則

  • 概念優先——先想一個諷刺、會反思的單一視覺概念（像紐約客封面），不是把新聞場景照畫。
  • 以 demo 圖為風格錨點——餵既有插畫當風格參考，鎖住「一眼可辨」的 The Pass 樣子。
  • 嚴守插畫指南——risograph 編輯插畫、出菜框＋桌鈴、人是主角；快訊不配圖。
  • 只發定稿——草稿不發。
