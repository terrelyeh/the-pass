編輯室的三個 Skill

整條出刊流程由三個 skill 接力完成，中間夾「人定稿」。點按鈕看每個 skill 在做什麼。

選題→寫稿→你定稿→發佈

選題

/selection-report
每期把全球來源抓回來、篩選評分，產出一份「選題報告」讓你拍板要寫哪幾篇。把「憑感覺挑題」變成有方法、可重複。
何時用
每期出刊的第一步。跟我說「跑選題 / 這期選什麼」。
流程
抓取 → 去重 → Haiku 子代理粗篩 → Claude 當總編五面向評分 → 庫存跨期競爭 → 選題報告（含三編輯切角）。
輸入輸出
來源庫 sources.ts→選題報告（網頁＋Obsidian）selected.json
特點
零 API key、在你本機 Claude Code 跑；跨期去重不重複撈／發。
看最新選題報告 ↗篩選機制設計 ↗

寫稿

/write-issue
把選題拍板的稿，寫成整期草稿（2 長文＋快訊＋1 提問）。四個編輯分身各用自己的聲音寫，AI 起草、人定稿。
何時用
選題拍板之後。跟我說「寫這期 / 把選的寫出來」。
流程
orchestrator 調度 4 個編輯分身（Mise／Passe／Fumet／總編）→ 各自載入 Soul＋Memory＋Voices → 抓原文查證 → 3 個人拍板 gate → 總編 7 項審核。
輸入輸出
selected.json→Obsidian 出刊草稿（待人定稿）
原則
事實不可扭曲（引號＝原文、數字＝原文）；聲音隔離、總編獨立審。
AI 編輯室架構 ↗編輯源頭（風格） ↗

發佈

/publish-issue
把你定稿的內容，做成 thepass.cc 的 issue 網頁，含長文配圖（紐約客式概念插畫）。
何時用
你離線潤定稿之後。跟我說「發佈這期」。
流程
讀定稿 → 長文配圖（概念優先＋以 demo 圖為風格錨點，nanobanana＋Codex 雙模型生圖 → 你挑圖）→ 渲染 issue 頁 → 更新配圖總覽 → 部署。
輸入輸出
定稿 md→issue-<date>.htmlimg/issue-<date>-N.png
原則
只發定稿；配圖嚴守插畫指南、快訊不配圖。
看首期內容 ↗配圖總覽 ↗插畫指南 ↗
