來源審核

/audit-sources

把一批候選來源評估後，自動收進選題 pipeline。一個指令，取代每次重講方法。

目的

同事推薦、或自己發現一批可能的來源時——快速、用同一套標準判斷「這值不值得收進 The Pass」，並把通過的自動落地。

The Pass 的內容好不好，七成決定在「料源」。但每次人工判斷容易飄、容易漏。這個 skill 把整套審核方法固化下來，讓每次評估都一致、可重複。

·做什麼（流程）

① 取出候選清單 — 網址 / 名稱 / 一篇列媒體的文章都行
↓
② 機械層：連通 + 活性 + 樣本 — 找出可用 feed、近期更新狀況、抓近 12 篇樣本標題（scripts/audit-feed.ts）
↓
③ 讀樣本，判內容走向 — Claude 依六個角度評估每個來源（下方）
↓
④ 出審核卡 + 收/不收建議 — 附建議的 stream / tier / category
↓
⑤ 你確認 → 自動落地 — 加進 sources.ts、重生來源頁、部署

Audit 的六個角度

① 連通
feed 抓得到嗎沒有可用 feed（紙本/JS-only/擋 bot）→ 不進 pipeline，可當編輯靈感。
② 活性
還在更新嗎最新一篇多久前、多久一篇。停更（>90 天）→ 不收。
③ 內容走向
是哪種食物內容（核心）類型分布（食譜 / 產業新聞 / 人物故事 / 文化觀點 / 科技 / 評論 / 趨勢 / 在地）＋ 調性（新鮮有趣 ↔ 乾的產業稿）＋ 餵哪位編輯（Mise 人物 / Passe 事實 / Fumet 觀點）。
④ 科技含量
食品科技 / AI 角度多不多輔助線——食品科技導向加分，但不是必要；純 AI / 通用科技不收。
⑤ 新鮮獨家
台灣讀者 / 台灣媒體還沒看過的程度越前沿、越奇、越有觀點 → 越高分。這是 The Pass 的價值核心：台灣沒有的新鮮事。
⑥ IG/影片
能不能延伸成 carousel / 短影音有畫面、有故事、有數字 ↔ 純文字產業稿。

→ 由以上推導出 分類建議：stream（A 飲食媒體 / B 食品科技·觀點）、tier（1 核心～5 觀察）、收（active）/ 待觀察（pending）/ 不收。

取材原則（判斷時的北極星）

  • TA 是台灣讀者，但刻意不收台灣來源——台灣飲食內容已太多，價值在「台灣沒有的新鮮事」：替台灣讀者開一扇窗，看世界上有趣的飲食新知 / 新聞 / 觀點。
  • 食物 / 餐飲 / 上下游為主，AI / 科技為輔——食品科技導向才收，純 AI / 通用科技不收。
  • 調性：新鮮感與發現的驚喜——不是乾的產業新聞。重人味、重觀點、反 AI slop。

不收的常見情況

  • 無可用 feed（紙本雜誌、JS-only、擋 bot）。
  • 停更（最新 > 90 天）。
  • 純 AI / 通用科技 / 新創，食物密度趨近 0。
  • 食譜為主——The Pass 不教做菜，食譜站適配弱。

怎麼用

下次直接說：
「幫我 audit 這些來源」+ 貼網址
或在 Claude Code 觸發 /audit-sources，就會自動跑完整套——不用再每次重講方法。

範例審核卡：
Foovo — 🟢 活躍（約 10 篇/週）
內容走向：90% 食品科技前沿（培養肉/精密發酵）+ 10% 報告導讀；偏技術但有發現感
餵：Passe 事實為主，偶有 Mise（創辦人專訪）｜科技高｜新鮮獨家高｜IG 潛力中
→ 收：Stream A · ai-food-tech · tier 3 · active
